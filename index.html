Three agent architectures. One Resolution Platform story.

The stack now includes three agent lineages, each with its own dialogue builder, its own taxonomy, and its own view of what "context" means for an agent. The product and AI leadership already know this. The question is which one earns the long-term investment and which ones collapse into shared infrastructure.

The 60 / 30 / 10 frame.

We think about any AI stack through a 60/30/10 lens. Applied to the Resolution Platform, that framing names which resolutions should cost $1.50 and which should cost $0.03. It also names which of the three inherited agent architectures earns the long-term investment.

60% Traditional code and database work. Ticket routing, knowledge lookup, status updates.
30% Rule-based logic. Policies, escalation thresholds, compliance gates.
10% Genuine AI work. Where an LLM earns its cost, and where a $1.50 resolution is the right price.

This is the conversation Zendesk's product and AI leadership are already having inside conference rooms. We have had it with other enterprise AI stacks and the methodology is written down.

The underlying methodology

Interpretable Context Methodology (ICM)

Folder Structure as Agent Architecture. Submitted to ACM TiiS. Repository public under MIT license. 52-member practitioner community.

It gives the vocabulary for layering agent context from identity through working artifacts, and it is the shortest path we know to rationalizing three acquired agent stacks into one coherent architecture.
